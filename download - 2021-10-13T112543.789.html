      Overview

      Imagine you are a designer, and you want a quick way of remembering all the hex codes for your favorite colors. If you had the colors (like "aqua", "beige", and "chartreuse") in an unsorted collection, you would have to start from the beginning and look through each item whenever you needed to look up a color. This searching process would take linear time (O(n)). If you had the colors sorted alphabetically, you could do a binary search, and that would take logarithmic time (O(log n)).

      The binary search method is pretty fast, but what if there was a way to retrieve the color you wanted in constant time (O(1)).

      In Python, a list of your colors would look like this:

      [("aqua", "#00FFFF"), ("beige", "#F5F5DC"), ("chartreuse", "#7FFF00")]

      As we said earlier, if this list of tuples was sorted alphabetically by the color name, we could use binary search to get the hex color value of a color in logarithmic time (O(log n)). But, we want a data structure for our colors that give us the ability to look up a colorâs hex value in constant time (O(1)).

      Hash Functions

      A hash function is a function where the input is any data, and the output is a number.

      So, in our case, we would want a hash function where I could input "aqua" and receive a number like 4 as the output.

      The requirements for a hash function are:

      • A hash function must be consistent (deterministic). Every time it receives the same input (like "aqua"), it must return the same output (like 4). If itâs not deterministic, it is not a hash function.
      • Different input data should return different numbers. For example, if the input "aqua" returns 4, then the input "beige" should not return 4.
      • A hash function must return numbers that are within a specific range.

      The reason the hash function should return different numbers when given different input data is to minimize collisions. (We provide more detail about collisions in a different objective.) When a hash function maps each different input data to a different number, it is called a perfect hash function. In practice, unless we know all the possible data inputs, we canât create a perfect hash function.

      A Simple Hash Table

      Whatâs the point of a hashing function? How does a function that maps data to a number get us any closer to a data structure with constant lookup time?

      We can combine a hash function with an array to achieve this. Imagine that we know we have five colors to store in our data structure. We can start with a list that has five empty slots.

      Now, we feed each of our colors into our hash function and use the number it returns as the list index where we store the associated hex code.

      Because the hash function is deterministic, if I want to look up the hex code associated with "beige", I merely input "beige" into the hash function and get 4 back. Now, I access the value stored at index 4 of the list and get "#F5F5DC".

      When you combine a hash function with an array, you get a data structure called a hash table (also known as a hash map or dictionary). In Python, hash tables are called dictionaries.

      Follow Along
      A Naive Hashing Function

      As we discussed above, hashing functions take an input (usually a string) and return an integer as the output. To convert a string into an integer, hashing functions operate on the individual characters that make up the string.

      In Python, we can encode strings into their bytes representation with the .encode() method (read more here). Once encoded, an integer represents each character.

      bytes_representation = "hello".encode()

for byte in bytes_representation:
    print(byte)

### Print Output
### 104
### 101
### 108
### 108
### 111

      Now that weâve converted our string into a series of integers, we can manipulate those integers in some way. For simplicityâs sake, we can use a simple accumulator pattern to get a sum of all the integer values.

      bytes_representation = "hello".encode()

sum = 0
for byte in bytes_representation:
    sum += byte

print(sum)

### Print Output
### 532

      We now have code that can convert a string into an integer. We need to house this logic in a function definition.

      def my_hashing_func(str):
    bytes_representation = str.encode()

    sum = 0
    for byte in bytes_representation:
        sum += byte

    return sum

      Our hashing function still has a limitation. We need our hashing function to return integers within a specific range. Remember, the purpose of our hashing function is to map our input data to an index value on a list.

      Modulo

      The modulo operator returns the remainder of dividing the left-hand operand and the right-hand operand. When the right-hand operand is greater than the left-hand operand, the modulo operator returns the value of the left-hand operand.

      5 % 2 # 1
7 % 4 # 3
3 % 6 # 3
3 % 7 # 3

      We can use the modulo operator in our hashing function to ensure that the integer the function returns is within a specific range.

      original_bytes_sum % size_of_list

      This operation ensures that the maximum return value is the last index in the list. Letâs modify our function to include this operation.

      def my_hashing_func(str, list_size):
    bytes_representation = str.encode()

    sum = 0
    for byte in bytes_representation:
        sum += byte

    return sum % list_size
      A Simple Hash Table

      Now, we have a working hashing function. We need to use it along with a list to store our color information.

      Here is the color information we would like to store in our hash table.

      ("aqua", "#00FFFF")
("beige", "#F5F5DC")
("chartreuse", "#7FFF00")
("deepskyblue", "#00BFFF")
("forestgreen", "#228B22")

      First, we need to initialize a list with five empty slots:

      my_list = [None] * 5

      Now, we need to use our hashing function to map the color name strings to the correct index in the list and then store the hex code value at that index. We can also retrieve the value using our hashing function.

      my_list = [None] * 5

my_list[my_hashing_func("aqua", len(my_list))] = "#00FFFF" # put value in list
print(my_list[my_hashing_func("aqua", len(my_list))]) # get value from list

### Print Output
### #00FFFF
      Performance

      The simple hash table we just created does not handle collisions (again, we cover more about collisions soon). It can search, insert, and delete all in constant time (O(1)).

      Constant time does not mean that it happens instantaneously. It still takes time to run the computations, of course. Constant time means that the computation takes the same amount of time regardless of the size of the hash table. If your hash table has one element, it takes the same amount of time to search for an item, as it would if your hash table had one trillion elements. The same is true for insertions and deletions.

      Why is this so? The reason is that we have the deterministic hashing function that tells us exactly which list index holds the data. Whether we are searching, inserting, or deleting, we pass the key to the hash function, and it returns the index where the associated value is stored (or should be stored). Then we can use that index to retrieve, insert, or delete.
